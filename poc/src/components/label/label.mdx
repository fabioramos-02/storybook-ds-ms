# Label — uso e acessibilidade

## Diferença de `.field-label`
`.field-label` (em `input.css`) já existia para o rótulo **dentro de um campo de formulário**
(`<label class="field-label" for="...">`). Este `.label` é a versão **standalone** — para
rotular conteúdo fora de um `<input>` (seção, valor, badge textual) — pedida como item próprio
na lista oficial de 13 componentes do Storybook. Os tokens (`tokens/label.json`) referenciam os
mesmos valores de fonte de `tokens/input.json`, então os dois ficam visualmente idênticos sem
duplicar números — só muda o uso (com/sem `for`/campo associado).

## Quando usar qual
- Rótulo de campo de formulário → `.field-label` (já existente, mantém `for`).
- Rótulo solto (texto descritivo de um valor, seção, ou par chave-valor) → `.label` (este).

## Acessibilidade
- `.required` é só visual (`*`); o requisito real de obrigatoriedade deve estar no atributo
  `required`/`aria-required` do campo associado, não só no asterisco — por isso aqui o `*` leva
  `aria-hidden="true"` no exemplo (a obrigatoriedade é comunicada pelo campo, não pelo label
  solto).
- `.is-disabled` é só indicação visual de contraste reduzido — não substitui `aria-disabled`
  no elemento real que está desabilitado.
