# Button — uso, acessibilidade e código

O Button é entregue como **classe CSS + HTML** (`.btn` + `styled` (paleta) + `type` (estilo) +
tamanho num `<button>` nativo). Fiel ao Figma (node `77:136152`, seção "Components").
Funciona em qualquer stack (PHP, Python, JS) sem framework.

## Eixos

- **styled** — paleta: `neutral` · `primary` · `secondary` · `tertiary`.
- **type** — estilo visual: `primary` (preenchido) · `secondary` (contornado) · `tertiary` (texto/ghost).
- **size** — `sm` (32px) · `md` (48px, padrão).

Formato shape: pill total (`border-radius: 9999px`) em todas as combinações.

Classes: `.btn .btn-{styled} .btn-{type} .btn-{size}` — onde `.btn-{styled}` é
`btn-neutral` / `btn-primary-palette` / `btn-secondary-palette` / `btn-tertiary-palette`
(prefixo `-palette` evita colisão com as classes de `type`, que usam `btn-primary` /
`btn-secondary` / `btn-tertiary` diretamente).

## Acessibilidade (eMAG + WCAG 2.1 AA)

Cada componente do DS-MS passa por testes nos moldes do
[USWDS](https://designsystem.digital.gov/components/button/accessibility-tests/).
Para o Button:

| Categoria | O que verificar | Como |
|---|---|---|
| **Geral** | Contraste do texto ≥ 4.5:1; alvo confortável | ANDI / Color Contrast Analyzer. Primário `#FFFFFF` sobre `#004F9F` ≈ 8:1 (passa). |
| **Zoom** | Botão e rótulo legíveis a 200% | Navegador, `Ctrl` + `+` |
| **Teclado** | Alcançável por `Tab`; foco **visível**; aciona com `Enter`/`Espaço` | `.btn:focus-visible` aplica `outline` de 2px |
| **Leitor de tela** | Anuncia papel "button" e estado desabilitado | NVDA / VoiceOver / Narrator |

Notas de implementação já presentes no `button.css`:
- `:focus-visible` com `outline` de 2px + `outline-offset` (foco perceptível).
- Estado desabilitado via atributo `disabled` (ou `aria-disabled="true"`), com `cursor: not-allowed`.
- Usa `<button>` nativo — papel/teclado vêm de graça do elemento.

> O painel **Accessibility** (addon axe-core) roda automaticamente em cada story.
> No CI, o mesmo check vira gate: violação = build vermelho.
