# Accordion — uso e acessibilidade

Implementado a partir da documentação pública de
[designsystem.ms.gov.br/accordion](https://www.designsystem.ms.gov.br/accordion/) — sem Figma de
origem (ver [`docs/08-proximos-passos.md`](../../../../docs/08-proximos-passos.md)).

## Decisão técnica: `<details>`/`<summary>` nativo, não Web Component
A spec pede: clique no header alterna expandido/colapsado, suporte a teclado, leitura por
screen reader e múltiplos itens abertos simultaneamente. **Tudo isso é nativo do HTML** via
`<details>`/`<summary>` — não há necessidade de JavaScript nem de Web Component. Modo
"exclusivo" (só um aberto por vez) é possível nativamente com `name` compartilhado entre
`<details>` (suporte recente nos browsers); fica como variante a aplicar quando necessário.

## Estrutura
```html
<details class="accordion">
  <summary>Título da seção</summary>
  <div class="accordion-body">Conteúdo — pode incluir sub-accordion.</div>
</details>
```

## Acessibilidade
- `<details>`/`<summary>` carrega semântica e estado (`aria-expanded` implícito) nativamente.
- `summary:focus-visible` com outline visível para navegação por teclado (Tab + Enter/Espaço).
- Ícone `::after` é puramente decorativo (não tem texto alternativo porque o estado já é
  comunicado pela semântica do elemento, não pelo ícone).
- `.is-disabled` remove interação via `pointer-events:none` — usar só quando o conteúdo
  realmente não está disponível, documentando o motivo ao lado.
